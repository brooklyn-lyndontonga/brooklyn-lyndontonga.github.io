What are the best practices associated with using classes vs. IDs?

This is something I am still learning to apply, and these are the things I have found as I research this topic:

Date: 05 August 2024

Here are some of my notes on what Classes & IDs are:

Classes:

  • Used for groups of elements: When you want to add the same style or property to multiple elements.
  • Multiple classes can be used: An element can have more than one class, so you can change more than one thing at a time.
  • Reusable: You can use the same class on many elements, so it can be used in different parts of your page.

IDs:

  • Use for unique elements: When you want to add a specific style or property to a single element, like a unique font or color.
  • One ID per page: An ID will only affect the single element it is assigned to on the page.
  • Single use: Only one element can have a particular ID on a page.

These are my notes on what best practice looks like:

  • 1. Use classes for styling and grouping: Apply styles or behaviors to groups of elements that share something in common, like headers, paragraphs, borders, padding, etc.
  • 2. Use IDs for unique elements: Identify and apply styles or behaviors to a single, specific element, like titles and headings.
  • 3. Keep IDs unique: Only use an ID once on a page.
  • 4. Combine classes when needed: An element can have multiple classes if it fits into multiple groups.

To summarize my learnings:

  • Classes: Group elements together.
  • IDs: Identify single, unique elements.
  • Use classes for multiple elements.
  • Use IDs for individual elements.
  • Ensure each ID is unique on a page.
